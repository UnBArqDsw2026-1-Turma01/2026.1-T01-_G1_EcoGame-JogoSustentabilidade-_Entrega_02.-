# 2.2. Módulo Notação UML – Modelagem Dinâmica

# Introdução 

Este artefato apresenta o Diagrama de estados do projeto ecoGame, modelado segundo a notação UML. O diagrama foi construído a partir dos requisitos funcionais levantados durante o Design Sprint e priorizados com a técnica MoSCoW, conforme documentado na [Especificação de Requisitos](../Projeto/requisitos.md).

O Diagrama de estado tem como objetivo representar os estados de um objeto sob a partir de ações do usuário (jogador), identificando os principais estados na qual aquele objeto pode estar.

## Metodologia 

Os diagramas foram elaborados utilizando o programa Draw.io usando a notação UML onde foram utilizados:

- **Estados** quadrados que demonstram como o objetivo está
- **Setas** que demonstram a transição entre estados
- **Esferas pretas** que demostram o inicio do processo
